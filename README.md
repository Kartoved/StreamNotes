# StreamNotes

**StreamNotes** — local-first приложение для заметок в формате ленты (как Twitter/Reddit threads). Все данные хранятся прямо в браузере — никаких серверов, никакой регистрации.

## Возможности

- **Local-first** — данные в браузере через CR-SQLite (Wasm). Работает без интернета
- **Богатый редактор** — TipTap: жирный, курсив, заголовки, блоки кода, цитаты, чеклисты, списки, ссылки
- **3-состояния чекбоксов** — unchecked → done → cancelled
- **Древовидные ветки** — заметки вложены друг в друга, как треды
- **Drag & Drop** — левая половина карточки = сиблинг, правая = дочерний
- **Обратные ссылки** — `[[заметка]]` создаёт backlink, виден в карточке
- **Вложения** — картинки, видео, файлы через 📎, drag-and-drop или вставку из буфера. Хранятся в OPFS, лимит 50 МБ
- **Галерея** — несколько изображений подряд отображаются сеткой, клик → лайтбокс
- **HEIC → JPEG** — автоконвертация Apple-фотографий при загрузке
- **Поиск** — полнотекстовый по содержимому заметок, работает с кириллицей
- **Теги** — `#тег` в тексте автоматически появляется в сайдбаре как фильтр
- **Календарь** — точки на днях с заметками, клик фильтрует ленту по дате
- **Свойства** — тип (tweet/task/document), статус (todo/doing/done/archived), дата; цветная подсветка карточки
- **Экспорт / Импорт** — бэкап всей базы в JSON одним кликом
- **Тёмная / светлая тема** — переключатель в хедере, сохраняется в localStorage
- **Ctrl+Enter** — отправить заметку без мыши
- **Виртуализация** — плавная лента даже при тысячах записей (@tanstack/react-virtual)
- **CRDT** — таблицы настроены как Conflict-Free Replicated Relations (база для будущей синхронизации)

## Стек

| Слой | Технология |
|---|---|
| UI | React + TypeScript |
| Сборка | Vite |
| БД | CR-SQLite Wasm + OPFS |
| Редактор | TipTap (ProseMirror) |
| Виртуализация | @tanstack/react-virtual |
| HEIC конвертация | heic2any |

## Структура

```
src/
├── components/
│   ├── Feed.tsx              # Лента: виртуализация, DnD, карточки, модалка, бэклинки
│   └── TiptapEditor.tsx      # Редактор + LexicalRender (read-only) + вложения + лайтбокс
├── db/
│   ├── schema.ts             # Схема SQLite + CRDT-настройки
│   ├── DBContext.tsx         # React-контекст для доступа к БД
│   └── hooks.ts              # useNotes — реактивный хук с рекурсивным CTE
├── utils/
│   ├── lexicalToTiptap.ts    # Конвертер Lexical AST ↔ TipTap JSON (обратная совместимость)
│   └── opfsFiles.ts          # Сохранение файлов в OPFS, resolveUrl, HEIC-конвертация
├── App.tsx                   # Хедер, сайдбар (поиск/календарь/теги), лейаут
├── editorTheme.css           # Стили TipTap-редактора
└── index.css                 # CSS-переменные тёмной/светлой темы, утилиты
```

## Запуск

```bash
npm install
npm run dev        # localhost:5173
npm run build      # production сборка
npm run preview    # превью сборки
npx tsc --noEmit  # проверка типов
```

> Требуются заголовки `Cross-Origin-Opener-Policy: same-origin` и `Cross-Origin-Embedder-Policy: require-corp` для работы SharedArrayBuffer (OPFS). Vite-конфиг уже их выставляет.

## Хранение данных

Контент заметок хранится как **Lexical AST JSON** в поле `content` таблицы `notes`. При открытии в редакторе конвертируется в TipTap JSON через `lexicalToTiptap`, при сохранении — обратно через `tiptapToLexical`. Это обеспечивает обратную совместимость со старыми записями.

Файлы-вложения хранятся в **OPFS** (`attachments/uuid.ext`), в БД только ссылка `attachment://uuid.ext`.
