# Архитектура Sheafy (Developer Guide)

В этом документе описаны ключевые архитектурные решения и технические детали проекта.

## 💾 Уровень данных (Data Layer)

### CR-SQLite (Vulcan)
Проект использует `@vlcn.io/crsqlite-wasm`. Это расширение для SQLite, которое превращает обычные таблицы в **Conflict-free Replicated Relations (CRRs)**.
- **Почему это важно?**: Любые данные, измененные локально, могут быть автоматически синхронизированы с другими устройствами по протоколу мульти-мастер репликации без конфликтов.
- **Хранилище**: База данных хранится в **Origin Private File System (OPFS)**. Это современный API браузера, обеспечивающий практически нативную производительность дисковых операций (быстрее, чем IndexedDB).

### Реактивная очистка и обновление
Слой `src/db/hooks.ts` реализует кастомные слушатели событий таблицы (`rx-tbl`), чтобы React-компоненты автоматически перерисовывались при изменении данных в БД.

## 🔐 Шифрование (Security Layer)

StreamNotes реализует **E2EE (End-to-End Encryption)** на стороне клиента.

### Управление ключами
- **Seed-фраза**: Приложение использует стандарт BIP-39 для генерации 12 слов.
- **Nostr-совместимость**: Ключи генерируются по аналогии с протоколом Nostr (secp256k1), что позволяет использовать ту же идентичность в других приложениях.
- **Мастер-ключ**: AES-ключ для шифрования данных извлекается из сид-фразы и хранится только в оперативной памяти (`CryptoContext.tsx`).

### Процесс шифрования
- **Данные**: Перед выполнением `INSERT` или `UPDATE` в SQLite, контент заметки (`content`) и её метаданные (`properties`) шифруются алгоритмом **AES-GCM**.
- **Поиск**: Полнотекстовый поиск реализован путем расшифровки данных на стороне клиента в runtime (виртуализированная лента позволяет делать это эффективно для видимых элементов). *В будущем планируется использование зашифрованных индексов.*
- **Миграция**: При обновлении приложения `App.tsx` проверяет наличие незашифрованных данных и автоматически переводит их в зашифрованный формат.

## 📝 Редактор (Rich Text)

### TipTap (ProseMirror)
Проект использует **TipTap**. Мы выбрали его за отличную экосистему расширений и гибкость в настройке Node Views.
- **Content State**: Каждая заметка сохраняется в формате JSON (TipTap JSON) в колонку `content`.
- **Markdown**: Редактор поддерживает основные Markdown-сокращения в режиме реального времени.
- **3-х статусные Checklists**: Реализовано 3 состояния: «Ожидает», «Выполнено» и «Отменено» через кастомное расширение.
- **Автодополнение Backlinks**: При вводе `[[` срабатывает плагин, который предлагает быстрый поиск по существующим заметкам.

## ⛓ Иерархия (Threading)

### Parent-Child модель
Заметки хранятся в плоской таблице, но визуально выстраиваются в дерево через `parent_id`.
- **Сортировка**: Используется `sort_key` (Fractional Indexing) для поддержки Drag-and-Drop.
- **Backlinks**: Редактор выявляет ссылки вида `note://UUID`. Данные попадают в таблицу `links`.

### Потоки (Feeds)
Система поддерживает несколько независимых лент (таблица `feeds`).
- Каждая заметка привязана к `feed_id`.
- Фиды имеют свои метаданные: название, цвет (используется для акцентов в UI) и аватар (хранится как data-URI или ссылка в OPFS, зашифрован).

## 🖼 Визуализация (Performance)

### Virtual List
Мы используем `@tanstack/react-virtual` для плавной работы с тысячами заметок.
- **Динамическое измерение**: Высота элементов измеряется "на лету", учитывая контент и вложения.
- **Зен-режим (Zen Mode)**: Для редактирования больших текстов используется `NoteModal`, который перекрывает весь интерфейс, создавая пространство для фокусировки.

## 🎨 Дизайн и UI

Приложение следует принципам **Glassmorphism**:
- Интенсивное использование `backdrop-filter: blur()`.
- Кастомные CSS переменные для управления цветовой схемой (палитра Slate/Blue/Accent).
- Легкие анимации переходов и индикаторы Drag-and-Drop.
