# Архитектура StreamNotes (Developer Guide)

В этом документе описаны ключевые архитектурные решения и технические детали проекта.

## 💾 Уровень данных (Data Layer)

### CR-SQLite (Vulcan)
Проект использует `@vlcn.io/crsqlite-wasm`. Это расширение для SQLite, которое превращает обычные таблицы в **Conflict-free Replicated Relations (CRRs)**.
- **Почему это важно?**: Любые данные, измененные локально, могут быть автоматически синхронизированы с другими устройствами по протоколу мульти-мастер репликации без конфликтов.
- **Хранилище**: База данных хранится в **Origin Private File System (OPFS)**. Это современный API браузера, обеспечивающий практически нативную производительность дисковых операций (быстрее, чем IndexedDB).

### Реактивная очистка и обновление
Слой `src/db/hooks.ts` реализует кастомные слушатели событий таблицы (`rx-tbl`), чтобы React-компоненты автоматически перерисовывались при изменении данных в БД (даже если изменения пришли извне или через прямой SQL запрос).

## 📝 Редактор (Rich Text)

### Lexical Engine
Мы выбрали **Lexical** (от Meta) за его модульность и производительность.
- **Content State**: Каждая заметка сохраняется в формате JSON AST (Abstract Syntax Tree) в колонку `content`. Это позволяет проводить глубокую аналитику контента на стороне БД или сервера в будущем.
- **Markdown**: Редактор поддерживает основные Markdown-сокращения в режиме реального времени.
- **3-х статусные Checklists**: В отличие от стандартных чек-боксов, здесь реализовано 3 состояния: «Ожидает», «Выполнено» и «Отменено» (через кастомный плагин `ThreeStateCheckListPlugin`).
- **Автодополнение Backlinks**: При вводе `[[` срабатывает `BacklinkPlugin`, который предлагает быстрый поиск по существующим заметкам.

## ⛓ Иерархия (Threading)

### Parent-Child модель
Заметки хранятся в плоской таблице, но визуально выстраиваются в дерево через `parent_id`.
- **Сортировка**: Используется `sort_key`. В текущей реализации это временный Fractional Indexing подход для поддержки Drag-and-Drop.
- **Backlinks**: При сохранении заметки, редактор выявляет ссылки вида `note://UUID`. Данные о связях попадают в таблицу `links`, что позволяет мгновенно находить «кто ссылается на эту заметку».

## 🖼 Визуализация (Performance)

### Virtual List
Так как StreamNotes предполагает длинные ленты бесконечных мыслей, мы используем `@tanstack/react-virtual`.
- **Измерение**: Высота элементов измеряется динамически, что критично для текстовых блоков разной длины.
- **Фокус**: При переходе по ссылкам `note://` приложение использует `virtualizer.scrollToIndex` для быстрой навигации.

## 🎨 Дизайн и UI

Приложение следует принципам **Glassmorphism**:
- Интенсивное использование `backdrop-filter: blur()`.
- Кастомные CSS переменные для управления цветовой схемой (палитра Slate/Blue/Accent).
- Легкие анимации переходов и индикаторы Drag-and-Drop.
